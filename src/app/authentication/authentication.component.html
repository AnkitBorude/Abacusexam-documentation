Authentication is a key component of our API, required for specific endpoints to ensure secure access and proper authorization. The system utilizes tokens, which must be included in the Authorization header in the format Bearer <token>. Separate tokens are issued for admin and student users, providing tailored access control based on the user's role. To obtain a token, users need to log in via the endpoint /api/v1/{student/admin}/login, where {student/admin} corresponds to the type of user.
Upon successful login, the response includes a token field containing the authentication token and a refreshToken. The authentication token is essential for accessing protected endpoints, while the refreshToken is used to renew the token when it expires.

Authorization Bearer <token>

The tokens issued are short-lived for enhanced security. When a token becomes invalid, it can be regenerated using the /api/v1/{student/admin}/token endpoint by providing the refresh token. This ensures uninterrupted access without requiring users to log in repeatedly.
Furthermore, some API responses and functionalities depend on the type of token provided. This mechanism enforces access control, ensuring that admin tokens grant higher privileges, while student tokens are limited to specific operations. This role-based authentication ensures that each user type interacts with the API in a controlled and secure manner.
